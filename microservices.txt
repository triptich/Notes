How do Microservcies Communicate With Each Other : 
There are 2 Different Types of Inter service Communication between microservices
1) Synchronous
2) Asynchronous 

Synchronous Communication :
The clients service waits for the response within a time limit. The possible solution is using Http protocol using via REST API for interservice communication.

Asynchronous Communication : The client Service doesnt wait for the response from another service. When the client microservice calls another microservice , thread is not blocked till the response comes from ther seerver.

The message producer service generates a message and sends the message to a message broker on a defined topic.
The message broker waits only for the acknowledgement from the message broker to know that the message is received by the broker.
The consuming service subscribes to the topic in the emssaging queue.All the messages belonging to the topic will be fed to the consuming system.
The message producer service and message consumer service dont even know each other.

Different messaging tools are based on AMQP(Advances MEssaging Queue Protocol)
Apache Kafka
RabbitMQ
ActivemQ
